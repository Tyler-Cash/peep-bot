## 2026-03-24 - IDOR in Event Management and Credential Exposure in Logs
**Vulnerability:** The `updateEvent` endpoint in `EventController` lacked an authorization check, allowing any authenticated user to modify any event. Additionally, sensitive `shareKey` credentials were being logged in plaintext in `PrepareAlbumOperation`.
**Learning:** Resource modification endpoints must always verify that the requester has the appropriate permissions (e.g., resource ownership or administrative privileges) relative to the specific resource being accessed. Furthermore, logging frameworks can inadvertently capture and store sensitive internal credentials if developers are not disciplined about log message content.
**Prevention:** Implement centralized or consistent authorization checks for all destructive or sensitive operations. For logging, perform regular audits of log statements in security-critical code paths to ensure no secrets or PII are being exposed. Use `Objects.equals` when comparing potentially null identifiers to avoid `NullPointerException`.
